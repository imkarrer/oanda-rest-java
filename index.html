Welcome to Rabun wrapper for OANDA REST API

Hi! We offer you to use our wrapper for the oanda rest api in java. We tried to implement most of the features of oanda rest api and provide you with a convenient java interface.

Below you will find a detailed description of working with our wrapper.

How to build it

We provide this library in the source code. This means that you have to create a jar file. As the build system we use maven. You can use maven to assemble a jar file.

Our library has the following dependencies 


<dependencies>
    <dependency>
        <groupId>com.mashape.unirest</groupId>
        <artifactId>unirest-java</artifactId>
        <version>1.3.20</version>
    </dependency>
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>4.11</version>
    </dependency>
    <dependency>
        <groupId>joda-time</groupId>
        <artifactId>joda-time</artifactId>
        <version>2.7</version>
    </dependency>
</dependencies>

To build the project use maven 3.x version. Execute the following commands: 

        
mvn package -Dmaven.test.skip=true

Note: maven 3.x doesn't work with intellij idea. Using terminal command line.

Further use of the received jar file in your project and don't forget to get some dependencies.

How to use it

To use the library is very simple! You need to use classes from the namespace rabun.oanda.rest.endpoints.

Currently the library includes 5 classes: OrderEndpoints, PositionEndpoints, RateEndpoints, TradeEndpoints, TransactionEndpoints.

  • OrderEndpoints - Contains methods for working with orders.
  • PositionEndpoints - Contains methods for working with positions. Includes methods to retrieve the list of positions and closing positions.
  • RateEndpoints - Methods for retrieving various information about the course, tools, etc..
  • TradeEndpoints - Methods for working with open transactions. Using these methods you can obtain a list of open trades, to modify them or close.
  • TransactionEndpoints - Information about committed transactions.

For more info see the docs files are in the project

The first thing you need to create an instance of the class OrderEndpoints. In the constructor pass the secret key and the account type (real or training) 

        
int accountId = 4905675;
String key = "019042a97b42ae9e1c0501e46cdb80fd-efe1760c46bfb65901418e8107e78827";
OrderEndpoints orderEndpoints = new OrderEndpoints(key, Endpoint.AccountType.practice);

More information you can find at source docs.
